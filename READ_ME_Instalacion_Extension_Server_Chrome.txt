-En este archivo se explica la forma de instalar Web Server for Chrome, una extensión que instala un pequeño servidor web en el navegador Chrome.

-Con ello se busca solucionar el error que ocurre cuando se carga un fichero JSON con $getJSON() relacionado con la CORS (Cross-origin resource sharing) policy debido a que se ha usado el IDE -> VSCode. 
(En el siguiente Sprint se buscara solucionar ese error usando el IDE WebStorm)



-Instalación:


-Link de instalación y activación de la extensión (NECESARIO para poder tener el panel abierto cada vez que se quiera ver los datos del JSON volcados):

 -> https://chrome.google.com/webstore/detail/web-server-for-chrome/ofhbbkphhbklhfoeikjpcbhemlocgigb/related



-Link de foro de ayuda -> https://es.stackoverflow.com/questions/215982/error-al-abrir-p%C3%A1gina-web-en-local-con-ajax-access-to-xmlhttprequest-at-file


-Pasos en el panel:

-Habilitar la extensión.

-Seleccionar carpeta de Planeta_Deporte

-En opciones avanzadas, habilitar la de "Set CORS headers"


***NOTA: Para que el servidor funcione, se debe dejar abierto el panel donde se configuran las anteriores opciones. Si no se hace esto, NO FUNCIONARÁ
